1、请阐述一下发明名称中所涉及的核心主题
核心主题是：为大型语言模型（Large Language Model，LLM）的指令微调（Instruction Tuning）阶段提供一种高质量的数据评估与筛选标准。
指令微调，即一种训练人工智能语言模型（如聊天机器人）的技术，需要使用大量且高质量的指令微调数据（通常为用户提问和模型回答的文本对），设计有效的数据评估与筛选标准的需求由此产生。针对这一问题，研究者们普遍认同数据越“多样”，微调所得的模型性能越好。但现有的数据评估与筛选标准往往不能较好分辨真正有价值的独特数据和低质量的乱码或错误数据（噪声），导致选出的数据虽然在数学上看起来很“稀疏”（分散），但实际上集合中混杂较多低质量数据。 为此，本发明提出了一种名为“语义凝聚态（Semantic Cohesion State）”的数据评估标准，其不仅能识别出数据的语义丰富程度，还能自动识别并降低那些语法错误、逻辑不通或与主题无关的“伪多样性”数据的权重，最终帮助研究者从海量数据源中挑选出真正高质量、高多样性的子集，用以提升模型的训练效果。


2、我们设计的新技术方案的设计动机或初衷
随着大型语言模型技术的发展，业界发现“数据的质量和多样性比单纯的数据数量更重要”。为了降低训练成本并提高模型性能，数据评估与筛选的核心任务是如何从海量的数据池中（例如包含数百万条互联网对话的数据集）挑选出最具代表性、最高效的一个子集来进行模型训练。因此当前的主要需求是：找到一种能够准确量化数据多样性的指标，且该指标必须与基于所筛选数据训练所得模型的最终性能高度正相关。
现有的数据多样性评估方法（主要基于几何稀疏性或嵌入距离）存在一个严重的逻辑漏洞：将“异常”误判为“多样性”，即认为在向量空间中距离多数数据样本最远的样本就是最有价值的（最“稀疏”的）。
产生原因： 这些方法仅仅考虑几何距离，并不理解文本的语义质量，这会导致两类数据噪声被错误地赋予高分：
1.内在低质噪声： 例如语法错误的句子（如“She don’t like apples”）或乱码。它们在向量空间中位置奇特，远离正常数据簇，容易被现有的方法判定为贡献数据集多样性的数据。
2.外在分布噪声：某些数据虽然具有较高质量，但其语义与整个数据集的核心语义关联极弱。包含较多类似数据的数据集，亦容易被现有方法判定为高质量数据集，而实际上其中隐藏大量损害模型性能的数据。


3、请结合2中的所确定的设计动机或初衷，用概括的语言，描述新技术方案是如何实现的
为解决“基于几何的数据评估指标无法区分数据噪声与真多样性”的问题，本新技术方案提出了一种“语义凝聚态”评估框架：“双视角评估 + 语义聚类加权”。 具体来说，我们不再单纯依赖几何距离，而是首先基于语义分布对数据进行聚类，随后利用全新的视角来共同决定一个数据的对数据集多样性的贡献权重：
内在视角（内在生成概率）： 针对每个数据（文本），利用大型语言模型计算该文本的内在生成概率，若其包含乱码或存在语法错误，模型生成该文本的概率将会很低，我们据此降低其权重，从而过滤低质噪声。
外在视角（空间凝聚度）： 针对聚类获得的每个数据簇内部，计算嵌入空间中所有数据到簇中心的距离。我们的方法认为距离中心越近的数据越能代表该数据簇的核心语义（凝聚力强），而距离较远的边缘孤立点会被分配较低权重。
结合两个视角所获得的权重，计算出所有数据加权后的信息熵（Entropy），以此作为精准的数据集多样性分数。通过这种手段，新技术方案建立了一种更为严谨的数据关系： 当一个数据样本既具备较高的语义质量（高生成概率），又具备良好的语义代表性（适当的空间位置）时，其对整个数据集多样性的贡献更容易被认可。


4、新技术方案具体是怎么做的
本新技术方案主要运行在可进行大型语言模型指令微调的服务器端或客户端，主要包含三个处理阶段：数据集语义聚类、语义凝聚态建模、基于熵的多样性量化。
一、数据集语义聚类
分析整个数据集的语义结构分布，将其划分为具有精细区别语义数据簇。对应图1中Semantic Cluster模块。
101、数据预处理。对指令微调数据集D=\{x_1,x_2,…,x_N \}中所有文本进行等长截断：大型语言模型LLM作为词元分词器，将所有文本截断为仅包含小于等于256个词元的新文本。这一步是为了消除过长文本对后续语义提取带来偏差的影响。
102、语义嵌入向量提取。用同一大型语言模型作为编码器E_"LLM" ，对截断后的每条文本x_i的进行语义编码，提取模型在所有词元位置输出的平均值，即一个高维语义向量，将其作为该文本的语义嵌入向量e_i。即 e_i=E_"LLM"  (x_i )。
103、自适应语义聚类。首先使用数据降维算法UMAP对102获得的所有语义嵌入向量进行降维处理，以保留主要语义结构，接着使用基于密度的层次聚类算法HDBSCAN对降维后的向量进行聚类，从而将D中对应的文本数据划分为若干个不相交的数据簇\{C_1,C_2,…,C_K \}。该聚类过程无需预设簇数量，自适应识别数据集潜在的语义结构，以达到较精确的划分效果。
（注：论文中提到设置最小聚类大小 M_min=30 为最优参数，可作为从属权利要求保护）。
二、语义凝聚态建模（核心阶段）
针对上述阶段划分好的每个数据簇中的每个文本数据计算其“语义凝聚权重”。对应图1中Semantic Cohesion State Modeling模块。
201、计算内在生成概率（Intrinsic Generation Probability）。利用上述阶段的同一大型语言模型LLM计算每个文本数据x_i中所有词元的条件生成概率并将其连乘获得整个文本的条件生成概率。为了避免长文本导致的数值衰减，将文本的生成概率的几何平均值作为该文本最终的内在生成概率。语法错误、乱码等低质内容的生成概率会显著较低。该过程可概括为p(x_i )=P_"LLM"  (x_i )，其中P_"LLM"  计算文本数据x_i的生成概率平均值，p(x_i )∈(0,1)可作为文本数据x_i在真实世界场景出现的概率的估计值。
202、计算空间距离权重（Spatial Weighting）。对于每个数据簇C_k，计算其中所有文本数据所对应的语义嵌入向量的平均值，以此作为该簇的中心点e_0，即e_0=1/|C_k |  ∑_(x_j∈C_k)▒e_j 。随后计算每个文本数据对应语义嵌入向量到该簇中心点的欧几里得距离，并应用高斯核函数（Gaussian-kernel）将每个文本数据的欧几里得距离转换为其空间距离权重，即w_i^dist=e^(-\|e_i-e_0 \|_2 )/(∑_(x_j∈C_k)▒e^(-\|e_j-e_0 \|_2 ) )×|C_k |。距离中心点越近的文本数据，其权重越高；距离越远的文本数据（边缘离群点），权重衰减越快。
203、合成语义凝聚权重。对于整个数据集中的每个文本数据，将其内在生成概率和空间距离权重相乘，得到该文本最终的语义凝聚有效权重，即(w_i ) ̂=p(x_i )×w_i^"dist" 。只有一条文本数据由大型语言模型生成的可能性高（质量好），同时在空间上具有较高凝聚性（非离群点），其语义凝聚权重才大。
三、基于熵的多样性量化
对应图1中Entropy-based Diversity 模块。
301、聚类级权重聚合。将每个数据簇内所有文本数据的语义凝聚权重相加，得到数据簇的总“语义凝聚态分数”，即S_(C_k )=∑_(x_i∈C_k)▒(w_i ) ̂ 。
302、全局分布归一化。将所有数据簇的语义凝聚态分数进行归一化处理，作为该数据集的全局概率分布，即P_(C_k )=S_(C_k )/(∑_(j=1)^K▒S_(C_j ) )。
303、计算香农熵（Shannon Entropy）： 对全局概率分布应用香农熵，得到最终的数据集多样性得分，即E_D=-∑_(k=1)^K▒〖P_(C_k )  log⁡〖P_(C_k ) 〗 〗。得分越高，代表数据集真正有效的多样性越好（且已排除了噪声干扰）。

5、新技术方案的亮点是什么
1. 首创的“双视角”评估机制： 本方案首次将“内在生成概率”（反映文本质量）与“外在空间位置”（反映语义代表性）结合，解决了传统方法只看空间距离而忽视文本质量的缺陷。
2. 有效剔除“伪多样性”： 能够自动识别并降权两类典型噪声：低质量文本（如语法错误）和分布外的高质量无关文本，防止这些噪声虚高多样性评分。
3. 与模型性能的超高相关性： 实验证明，使用该指标筛选出的数据，其训练出的模型性能与指标分数的相关性高达 0.985（在Qwen3模型上），远超现有最先进技术（NovelSum）及传统词汇指标（TTR）。
4. 符合信息熵的对数增长规律： 本方案计算出的多样性曲线呈现对数增长并逐渐饱和，符合信息论规律，能够敏锐捕捉到数据量增加后的“信息冗余”现象。
5. 计算效率的平衡： 相比于部分极度耗时的最先进方法（如NovelSum），本方案在保持最高准确率的同时，计算速度提升了约 9-10 倍（参考论文Table 5，SCS vs NovelSum），具备实际落地的可行性。
